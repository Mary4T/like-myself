# 任務管理應用程式 - 功能與架構說明

## 一、應用程式概述

這是一個功能豐富的任務管理系統，專注於支援複雜的專案任務管理、重複性任務追蹤、以及動態任務生成。應用程式採用 React 框架開發，使用 localStorage 進行資料持久化，無需後端伺服器即可運行。

## 二、核心功能

### 1. 任務樹狀結構管理

**功能描述：**
- 支援多層級任務結構（A/B/C/D/E 層級）
- 任務可以包含無限層級的子任務
- 支援拖曳排序和重新組織任務層級
- 任務可以標記為隱藏（isHidden）

**技術實現：**
- 使用遞迴結構儲存任務樹（`children` 陣列）
- 層級自動計算（父任務層級 + 1）
- 使用 `@hello-pangea/dnd` 實現拖曳功能

### 2. 任務重複性系統（Repeat System）

**功能描述：**
這是一個複雜且完整的重複任務管理系統，支援任務按設定的間隔自動重置。

**核心特性：**

#### 2.1 重複設定
- **時間單位**：支援「分鐘」和「天」兩種最小單位
- **間隔設定**：可設定重複間隔（如每 3 天、每 30 分鐘）
- **基準點**：可選擇以「開始日期」或「截止日期」作為重置基準
- **重置規則**：
  - 任務完成狀態重置為 `pending`
  - 進度重置為 0
  - 子任務遞迴重置（保留原始狀態類型）
  - 暫停狀態的任務跳過重置

#### 2.2 重置引擎（Reset Engine）
系統在三個時機點檢查並執行重置：
1. **啟動時檢查**：應用程式啟動時檢查所有重複任務
2. **定時輪詢**：每分鐘檢查一次（針對分鐘級重複）
3. **標籤切換**：進入任務詳情或甘特圖視圖時即時檢查

**重置邏輯：**
```javascript
// 核心重置函數：resetTaskTreeIfNeeded
- 計算當前時間窗口（windowStart）
- 判斷是否需要重置（shouldResetTask）
- 執行 backfill（補記）邏輯
- 遞迴重置子任務樹
```

#### 2.3 重複日誌系統（Repeat Log）
- **日誌記錄**：每天/每分鐘記錄任務完成狀態和進度
- **日誌保留**：無限期保留（使用者可手動刪除）
- **Backfill 機制**：最多補記最近 7 天的日誌（避免一次性回補過多）
- **日誌視覺化**：
  - 日誌列表：顯示最近 7-30 天的記錄
  - 日誌分析：包含完成率統計、趨勢圖表
  - 手動刪除：可刪除特定日期的日誌記錄

**日誌資料結構：**
```javascript
repeatLog: {
  "2025-01-15": {
    completed: true,
    progress: 100,
    maxProgress: 100
  }
}
```

### 3. 文字膠囊動態任務生成系統

**功能描述：**
這是一個創新的任務生成系統，允許使用者透過「文字膠囊」和「任務模板」批量生成結構化的子任務。

#### 3.1 文字膠囊（Text Capsules）
- 使用者可以為任務建立多個文字膠囊（如：「書房」、「客廳」、「臥室」）
- 每個膠囊可以獨立編輯和刪除
- 膠囊狀態按任務獨立儲存

#### 3.2 膠囊任務模板設計器（Capsule Task Template Designer）

**核心功能：**

1. **模板結構設計**
   - 支援多層級任務結構設計
   - 任務名稱可包含固定文字和膠囊佔位符
   - 使用 `contentEditable` 實現混合內容編輯（文字 + 膠囊標籤）

2. **膠囊佔位符系統**
   - 顯示為藍色標籤樣式（「膠囊」）
   - 可在任務名稱中任意位置插入
   - 支援在文字間拖曳移動膠囊位置
   - 生成任務時，佔位符會被實際膠囊文字替換

3. **任務拖曳重組**
   - 任務節點可拖曳成為其他任務的子任務
   - 拖曳到任務上半部：成為同層級任務
   - 拖曳到任務下半部：成為子任務
   - 拖曳失敗時自動回退到原位置

4. **模板載入**
   - 可從預設模板載入（使用「任務模板」功能的所有模板）
   - 載入時自動計算任務層級（忽略模板原有層級）

5. **任務生成**
   - 每個膠囊生成一組獨立的任務樹
   - 自動替換模板中的膠囊佔位符
   - 生成的任務標記來源任務和膠囊 ID

**技術實現亮點：**
- 使用 `nameParts` 陣列結構儲存混合內容（文字 + 膠囊）
- `parseNamePartsFromElement` 和 `renderNamePartsToElement` 實現 DOM 與資料結構的雙向轉換
- 使用 `document.createTreeWalker` 解析 `contentEditable` 元素
- 膠囊拖曳使用 HTML5 Drag and Drop API，配合 `document.caretRangeFromPoint` 定位插入點

### 4. 任務模板系統

**功能描述：**
- 預設內建「生活環境整理」模板
- 模板包含完整的任務樹結構和描述
- 可從模板快速建立專案任務
- 模板支援「膠囊標記」（hasCapsules），標記需要填入膠囊的任務

### 5. 任務詳情管理

**功能描述：**
- **富文本編輯器**：使用 ReactQuill 實現任務描述編輯
- **日期時間選擇器**：開始日期、截止日期、時間設定
- **提前提醒系統**：可設定多個提醒（支援年/月/日/小時/分鐘單位）
- **地點選擇器**：整合 Google Maps API
- **圖示選擇器**：預設圖示庫 + 自訂圖示上傳
- **手繪畫布**：DrawingCanvas 元件支援手繪功能
- **超連結管理**：可新增多個外部連結
- **集氣條（Cheer Up）**：任務激勵系統，可新增集氣項目並追蹤進度

### 6. 任務視圖系統

#### 6.1 任務詳情視圖（Details）
- 完整的任務資訊編輯
- 可自訂布局（布局編輯模式）
- 支援拖曳重新排列區塊順序

#### 6.2 任務概覽視圖（Overview）
- 任務列表顯示
- 多條件篩選（層級、狀態、優先級、標籤）
- 多欄位排序（標題、創建時間、截止日期、優先級、層級、狀態）
- 批量選擇和操作

#### 6.3 標籤視圖（Tags）
- 任務標籤管理
- 標籤顏色自訂
- 按標籤篩選任務

#### 6.4 甘特圖視圖（Gantt）
- 時間軸視覺化
- 支援日/週/月三種縮放級別
- 任務條顯示開始和結束時間
- 日視圖支援 hover 預覽和編輯
- 可調整欄寬
- 任務顯示模式（預設/層級/優先級/自訂）

#### 6.5 日曆視圖（Calendar）
- 月曆顯示
- 任務按日期分組顯示
- 點擊日期查看當日任務

### 7. 通知系統

**功能描述：**
- 瀏覽器原生通知 API
- 任務到期提醒
- 提前提醒（可設定多個）
- 通知歷史記錄
- 通知權限管理

### 8. 資料持久化

**功能描述：**
- 所有資料儲存在 `localStorage`
- 自動儲存機制（使用 debounce 優化）
- 資料遷移機制（為舊資料補齊新欄位）

**儲存的資料：**
- `projectTasks`：任務樹資料
- `taskCapsuleStates`：膠囊狀態（按任務）
- `projectTaskTemplates`：任務模板
- `taskTags`：標籤資料
- `projectNotificationHistory`：通知歷史
- `ganttTaskDisplayMode`、`ganttColumnWidth`：使用者偏好設定

### 3. 狀態管理架構

**狀態管理方式：**
- 使用 React Hooks（`useState`、`useEffect`、`useMemo`、`useCallback`、`useRef`）
- 無使用 Redux 或 Context API（所有狀態在元件內部管理）
- 狀態持久化透過 `localStorage`

**主要狀態分類：**

1. **任務資料狀態**
   - `tasks`：任務樹根陣列
   - `selectedTask`：當前選中的任務

2. **UI 狀態**
   - `activeTab`：當前視圖標籤
   - `isLayoutEditing`：布局編輯模式
   - `toolbarCollapsed`：工具列摺疊狀態

3. **功能狀態**
   - `taskCapsuleStates`：膠囊狀態（按任務 ID 索引）
   - `templates`：任務模板列表
   - `taskTags`：標籤列表
   - `activeNotifications`：活動通知列表

4. **視圖狀態**
   - `calendarDate`：日曆視圖當前月份
   - `ganttZoom`：甘特圖縮放級別
   - `ganttTaskDisplayMode`：甘特圖顯示模式
   - `searchTerm`、`levelFilter`、`statusFilter` 等：篩選和排序狀態

### 5. 核心演算法與邏輯

#### 5.1 任務樹遍歷
- **遞迴更新**：`updateTaskTree` 函數遞迴遍歷任務樹進行更新
- **任務查找**：`findTaskById` 遞迴查找指定 ID 的任務
- **層級計算**：根據父任務層級自動計算子任務層級

#### 5.2 重複任務重置邏輯
- **時間窗口計算**：`getWindowStart` 根據基準點和間隔計算當前時間窗口
- **重置判斷**：`shouldResetTask` 判斷任務是否需要重置
- **遞迴重置**：`resetTaskTreeIfNeeded` 遞迴重置任務及其子任務
- **Backfill 機制**：自動補記錯過的日誌記錄（最多 7 天）

#### 5.3 膠囊任務生成邏輯
- **模板轉換**：將模板結構轉換為實際任務結構
- **佔位符替換**：將 `nameParts` 中的膠囊佔位符替換為實際膠囊文字
- **層級繼承**：生成的任務繼承模板中的層級設定

#### 5.4 內容解析與渲染
- **混合內容解析**：`parseNamePartsFromElement` 解析 `contentEditable` 中的文字和膠囊標籤
- **混合內容渲染**：`renderNamePartsToElement` 將 `nameParts` 陣列渲染為 DOM 元素

### 6. 效能優化策略

1. **Debounce 機制**：使用 Lodash debounce 延遲 localStorage 寫入
2. **useMemo**：計算衍生資料（如篩選後的任務列表、日曆任務、甘特圖任務）
3. **useCallback**：穩定函數引用，避免不必要的重新渲染
4. **條件渲染**：只在需要時渲染複雜元件（如甘特圖、日曆）

### 7. 使用者體驗設計

1. **響應式布局**：支援不同螢幕尺寸
2. **拖曳視覺回饋**：拖曳時顯示插入指示線
3. **即時儲存**：所有變更自動儲存到 localStorage
4. **錯誤處理**：try-catch 包裹所有 localStorage 操作
5. **資料遷移**：自動為舊資料補齊新欄位

## 四、特別設計與複雜功能

### 1. 重複任務系統的完整實現

這是一個非常完整的重複任務管理系統，包含：
- 多時間單位支援（分鐘/天）
- 自動重置引擎（啟動/定時/切換檢查）
- 完整的日誌系統（記錄、補記、視覺化、分析）
- 遞迴子任務重置
- 暫停任務的特殊處理

### 2. 文字膠囊 + 模板設計器的混合編輯系統

這是一個創新的任務生成系統：
- **混合內容編輯**：在 `contentEditable` 中同時編輯文字和可拖曳的膠囊標籤
- **DOM 與資料結構雙向轉換**：實現了複雜的解析和渲染邏輯
- **膠囊拖曳定位**：使用 `document.caretRangeFromPoint` 精確定位插入點
- **任務模板可視化設計**：支援拖曳重組任務結構

### 3. 可自訂布局系統

- 支援拖曳重新排列任務詳情頁面的區塊
- 布局狀態儲存在任務資料中
- 提供布局編輯模式

### 4. 多視圖系統

- 五種不同的任務視圖（詳情/概覽/標籤/甘特圖/日曆）
- 每種視圖都有獨立的狀態管理和優化
- 甘特圖支援三種縮放級別和自訂顯示模式

## 五、技術亮點總結

1. **無後端架構**：完全基於 localStorage 的單頁應用
2. **複雜狀態管理**：使用 React Hooks 管理大量狀態，無需額外狀態管理庫
3. **遞迴資料結構**：任務樹的遞迴處理和更新
4. **混合內容編輯**：contentEditable + 自訂元素拖曳的複雜實現
5. **時間處理邏輯**：複雜的重複任務重置和日誌補記邏輯
6. **視覺化功能**：甘特圖和日曆的 SVG 渲染
7. **效能優化**：使用 memoization 和 debounce 優化效能
