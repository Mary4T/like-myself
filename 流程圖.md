# 任務管理應用程式 - 流程圖

## 1. 應用程式啟動流程

```mermaid
flowchart TD
    A[應用啟動] --> B[載入 localStorage 資料]
    B --> C{是否有保存的任務?}
    C -->|是| D[解析 JSON 資料]
    C -->|否| E[建立預設根任務]
    D --> F[資料遷移檢查]
    E --> F
    F --> G[執行重複任務重置檢查]
    G --> H[啟動定時輪詢 每分鐘檢查]
    H --> I[渲染任務樹]
    I --> J[應用就緒]
```

## 2. 重複任務重置流程

```mermaid
flowchart TD
    A[觸發重置檢查] --> B{觸發時機}
    B -->|啟動時| C[performResetCheck startup]
    B -->|定時輪詢| D[performResetCheck interval]
    B -->|標籤切換| E[performResetCheck tab]
    C --> F[遍歷所有任務]
    D --> F
    E --> F
    F --> G{任務有啟用重複?}
    G -->|否| H[跳過此任務]
    G -->|是| I[計算時間窗口 windowStart]
    I --> J{需要重置? shouldResetTask}
    J -->|否| H
    J -->|是| K[執行重置 resetTaskTreeIfNeeded]
    K --> L[重置任務本身]
    L --> M[記錄當前窗口日誌]
    M --> N[執行 Backfill 補記]
    N --> O{有子任務?}
    O -->|是| P[遞迴重置子任務]
    O -->|否| Q[更新 lastResetAt]
    P --> R{子任務狀態是暫停?}
    R -->|是| S[跳過此子任務]
    R -->|否| T[重置子任務狀態和進度]
    T --> Q
    S --> Q
    Q --> H
    H --> U{還有其他任務?}
    U -->|是| F
    U -->|否| V[儲存到 localStorage]
    V --> W[完成]
```

## 3. 膠囊任務生成流程

```mermaid
flowchart TD
    A[使用者點擊生成子任務] --> B[獲取當前任務膠囊狀態]
    B --> C{是否有膠囊?}
    C -->|否| D[提示需要先新增膠囊]
    C -->|是| E{是否有自定義模板?}
    E -->|否| F[提示需要設計模板]
    E -->|是| G[取得未生成的膠囊列表]
    G --> H[遍歷每個膠囊]
    H --> I[載入模板結構]
    I --> J[轉換模板為任務 convertTemplateToTask]
    J --> K[替換膠囊佔位符]
    K --> L{模板任務有 nameParts?}
    L -->|是| M[遍歷 nameParts]
    L -->|否| N[使用原始標題]
    M --> O{是膠囊類型?}
    O -->|是| P[替換為膠囊文字]
    O -->|否| Q[保留原始文字]
    P --> R[組合任務名稱]
    Q --> R
    N --> R
    R --> S[建立任務物件]
    S --> T{有子任務?}
    T -->|是| U[遞迴轉換子任務]
    T -->|否| V[標記為已生成任務]
    U --> V
    V --> W[加入任務樹]
    W --> X{還有其他膠囊?}
    X -->|是| H
    X -->|否| Y[更新膠囊狀態 generatedCapsuleIds]
    Y --> Z[儲存到 localStorage]
    Z --> AA[重新渲染任務樹]
    AA --> BB[完成]
```

## 4. 任務新增/編輯流程

```mermaid
flowchart TD
    A[使用者操作] --> B{操作類型}
    B -->|新增任務| C[建立新任務物件]
    B -->|編輯任務| D[取得選中任務]
    B -->|刪除任務| E[確認刪除]
    C --> F[設定任務屬性]
    D --> G[更新任務屬性]
    E --> H[從任務樹移除]
    F --> I[計算任務層級]
    G --> I
    I --> J[更新任務樹 updateTaskTree]
    J --> K[遞迴查找父任務]
    K --> L[更新任務或子任務]
    L --> M[儲存到 localStorage]
    M --> N[重新渲染 UI]
    H --> M
    N --> O[完成]
```

## 5. 膠囊模板設計流程

```mermaid
flowchart TD
    A[啟用文字膠囊功能] --> B[初始化膠囊狀態]
    B --> C[開啟模板設計器]
    C --> D{載入模板?}
    D -->|是| E[選擇預設模板]
    D -->|否| F[建立空白模板]
    E --> G[轉換模板結構]
    F --> H[新增根任務]
    G --> I[計算任務層級]
    H --> I
    I --> J[渲染模板任務樹]
    J --> K[使用者編輯任務名稱]
    K --> L{插入膠囊?}
    L -->|是| M[點擊 +新增膠囊]
    L -->|否| N[直接編輯文字]
    M --> O[在游標位置插入膠囊標籤]
    N --> P[更新 nameParts]
    O --> P
    P --> Q[解析 contentEditable parseNamePartsFromElement]
    Q --> R[更新模板狀態]
    R --> S{拖曳任務?}
    S -->|是| T[拖曳任務節點]
    S -->|否| U[儲存模板]
    T --> V{拖曳到目標位置}
    V -->|上半部| W[成為同層級任務]
    V -->|下半部| X[成為子任務]
    W --> Y[更新任務樹結構]
    X --> Y
    Y --> Z[重新計算層級]
    Z --> U
    U --> AA[儲存到 taskCapsuleStates]
    AA --> BB[完成]
```

## 7. 重複日誌更新流程

```mermaid
flowchart TD
    A[任務狀態變更] --> B{變更類型}
    B -->|完成狀態| C[更新任務 completed]
    B -->|進度變更| D[更新任務 progress]
    C --> E[取得當前時間窗口]
    D --> E
    E --> F[計算窗口鍵值 dateKey]
    F --> G[取得或建立日誌記錄]
    G --> H[更新日誌記錄]
    H --> I[更新 completed 狀態]
    I --> J[更新 progress 和 maxProgress]
    J --> K[儲存到 repeatLog]
    K --> L[更新任務物件]
    L --> M[儲存到 localStorage]
    M --> N[更新日誌顯示]
    N --> O[完成]
```

## 8. 膠囊拖曳流程（在任務名稱中）

```mermaid
flowchart TD
    A[使用者開始拖曳膠囊] --> B[設定 draggable=true]
    B --> C[觸發 dragstart 事件]
    C --> D[儲存拖曳的膠囊元素]
    D --> E[使用者移動滑鼠]
    E --> F[觸發 dragover 事件]
    F --> G[取得滑鼠位置]
    G --> H[使用 caretRangeFromPoint 定位]
    H --> I{找到插入點?}
    I -->|是| J[建立 Range 物件]
    I -->|否| K[隱藏插入指示器]
    J --> L[顯示插入指示器藍線]
    L --> M[使用者放開滑鼠]
    M --> N[觸發 drop 事件]
    N --> O[阻止預設行為]
    O --> P[移除原始膠囊元素]
    P --> Q[在插入點插入新膠囊元素]
    Q --> R[解析 contentEditable parseNamePartsFromElement]
    R --> S[更新 nameParts 陣列]
    S --> T[更新模板狀態]
    T --> U[儲存到 localStorage]
    U --> V[完成]
    K --> M
```

## 9. 任務拖曳重組流程

```mermaid
flowchart TD
    A[使用者開始拖曳任務] --> B[觸發 onDragStart]
    B --> C[儲存拖曳任務資訊 draggedTaskInfo]
    C --> D[設定 draggable 屬性]
    D --> E[使用者移動到目標任務]
    E --> F[觸發 onDragOver]
    F --> G[計算滑鼠位置]
    G --> H{在目標任務上半部?}
    H -->|是| I[顯示上方插入線]
    H -->|否| J[顯示下方插入線]
    I --> K[設定拖曳模式為 sibling]
    J --> L[設定拖曳模式為 child]
    K --> M[使用者放開滑鼠]
    L --> M
    M --> N[觸發 onDrop]
    N --> O[取得目標任務]
    O --> P{目標是拖曳任務的子任務?}
    P -->|是| Q[取消操作 回退到原位置]
    P -->|否| R{拖曳模式}
    R -->|sibling| S[移動為同層級任務]
    R -->|child| T[移動為子任務]
    S --> U[從原位置移除任務]
    T --> U
    U --> V[插入到新位置]
    V --> W[重新計算層級 adjustTaskLevels]
    W --> X[更新任務樹]
    X --> Y[儲存到 localStorage]
    Y --> Z[重新渲染任務樹]
    Z --> AA[完成]
    Q --> AB[恢復原位置]
    AB --> AA
```

## 10. 資料持久化流程

```mermaid
flowchart TD
    A[任務資料變更] --> B[觸發更新函數]
    B --> C[更新 React 狀態]
    C --> D[使用 debounce 延遲]
    D --> E{延遲時間到?}
    E -->|否| F[繼續等待]
    E -->|是| G[準備儲存資料]
    G --> H[序列化為 JSON]
    H --> I[嘗試寫入 localStorage]
    I --> J{寫入成功?}
    J -->|是| K[儲存完成]
    J -->|否| L[記錄錯誤]
    F --> D
    L --> M[顯示錯誤提示]
    K --> N[完成]
    M --> N
```

## 11. 通知系統流程

```mermaid
flowchart TD
    A[任務到期檢查] --> B[遍歷所有任務]
    B --> C{任務有設定提醒?}
    C -->|否| D[跳過]
    C -->|是| E[計算提醒時間]
    E --> F{到達提醒時間?}
    F -->|否| D
    F -->|是| G{已發送過通知?}
    G -->|是| H[檢查通知歷史]
    G -->|否| I[請求通知權限]
    H --> J{在同一個時間窗口?}
    J -->|是| D
    J -->|否| I
    I --> K{權限已授予?}
    K -->|否| L[提示需要權限]
    K -->|是| M[建立通知]
    M --> N[發送通知]
    N --> O[記錄到通知歷史]
    O --> P[更新通知狀態]
    P --> Q[儲存到 localStorage]
    Q --> R[完成]
    D --> S{還有其他任務?}
    S -->|是| B
    S -->|否| R
```

## 12. 模板載入流程

```mermaid
flowchart TD
    A[使用者選擇載入模板] --> B[選擇模板 ID]
    B --> C[從 templates 陣列查找]
    C --> D{找到模板?}
    D -->|否| E[顯示錯誤]
    D -->|是| F[取得模板 tree 結構]
    F --> G[轉換模板樹 convertTemplateTree]
    G --> H[遞迴處理每個節點]
    H --> I[建立模板任務物件]
    I --> J[計算任務層級 根據深度]
    J --> K[轉換標題為 nameParts]
    K --> L{有子任務?}
    L -->|是| M[遞迴處理子任務]
    L -->|否| N[加入模板任務陣列]
    M --> N
    N --> O[更新 capsuleTaskTemplate]
    O --> P[儲存到 taskCapsuleStates]
    P --> Q[重新渲染模板設計器]
    Q --> R[完成]
```

## 使用說明

這些流程圖使用 **Mermaid** 語法編寫，可以直接在以下平台使用：

1. **GitHub**：在 Markdown 文件中直接貼上，GitHub 會自動渲染
2. **GitLab**：同樣支援 Mermaid
3. **其他平台**：需要 Mermaid 支援的編輯器或網站

### 在 GitHub 中使用：

1. 建立或編輯 `.md` 文件
2. 貼上上述任何一個流程圖的程式碼
3. GitHub 會自動渲染成流程圖

### 範例：

```markdown
## 應用程式啟動流程

\```mermaid
flowchart TD
    A[應用啟動] --> B[載入 localStorage 資料]
    ...
\```
```

注意：在實際使用時，反斜線 `\` 不需要，這裡只是為了顯示語法。
