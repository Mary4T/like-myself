## 一、應用程式概述

### 1.1 應用程式定位

這是一個**功能豐富的任務管理系統**，專注於支援：
- 複雜的專案任務管理（多層級任務樹）
- 重複性任務追蹤（自動重置系統）
- 動態任務生成（文字膠囊 + 模板系統）
- 多視圖展示（詳情/概覽/標籤/甘特圖/日曆）

### 1.2 技術特點

- **無後端架構**：完全基於 `localStorage` 的單頁應用（SPA）
- **純前端實現**：使用 React 18.2.0，無需伺服器
- **資料持久化**：所有資料自動儲存到瀏覽器本地
- **即時更新**：使用 React Hooks 管理狀態，即時反映變更

### 1.3 應用程式路由

```
/          → Home.js（首頁 - 簡單任務列表）
/projects  → ProjectList.js（主要任務管理系統）
```

### 2.2 狀態管理架構

**管理方式：**
- 使用 **React Hooks**（`useState`、`useEffect`、`useMemo`、`useCallback`、`useRef`）
- **無使用 Redux 或 Context API**（所有狀態在元件內部管理）
- **狀態持久化**：透過 `localStorage` 自動儲存

**主要狀態分類：**

#### 2.2.1 任務資料狀態
- `tasks`：任務樹根陣列（遞迴結構）
- `selectedTask`：當前選中的任務
- `expandedItems`：展開的任務 ID Set

#### 2.2.2 UI 狀態
- `activeTab`：當前視圖標籤（'details' | 'overview' | 'tags' | 'gantt' | 'calendar'）
- `isLayoutEditing`：布局編輯模式
- `toolbarCollapsed`：工具列摺疊狀態
- `showAddModal`：新增任務模態框
- `showTemplateModal`：模板選擇模態框

#### 2.2.3 功能狀態
- `taskCapsuleStates`：膠囊狀態（按任務 ID 索引）
- `templates`：任務模板列表
- `taskTags`：標籤列表（含顏色）
- `activeNotifications`：活動通知列表
- `notificationHistory`：通知歷史記錄

#### 2.2.4 視圖狀態
- `calendarDate`：日曆視圖當前月份
- `ganttZoom`：甘特圖縮放級別（'day' | 'week' | 'month'）
- `ganttTaskDisplayMode`：甘特圖顯示模式（'default' | 'level' | 'priority' | 'custom'）
- `ganttColumnWidth`：甘特圖欄寬（使用者偏好）
- `searchTerm`、`levelFilter`、`statusFilter`、`priorityFilter`、`tagFilter`：篩選和排序狀態

### 2.3 資料持久化機制

**儲存的資料鍵值：**
- `projectTasks`：任務樹資料
- `taskCapsuleStates`：膠囊狀態（按任務）
- `projectTaskTemplates`：任務模板
- `taskTags`：標籤資料
- `projectNotificationHistory`：通知歷史
- `ganttTaskDisplayMode`、`ganttColumnWidth`：使用者偏好設定

**儲存策略：**
- 使用 **Lodash debounce** 延遲寫入（避免頻繁寫入）
- 所有 `localStorage` 操作包裹在 `try-catch` 中
- 自動資料遷移機制（為舊資料補齊新欄位）

## 三、核心功能模組

### 3.1 任務樹狀結構管理

**功能描述：**
- 支援**多層級任務結構**（A/B/C/D/E 層級，理論上無限層級）
- 任務可以包含無限層級的子任務（遞迴結構）
- 支援**拖曳排序**和重新組織任務層級
- 任務可以標記為**隱藏**（`isHidden`），不顯示在樹狀圖中

**技術實現：**
- 使用**遞迴結構**儲存任務樹（`children` 陣列）
- 層級自動計算（父任務層級 + 1）
- 使用 `@hello-pangea/dnd` 實現拖曳功能
- 拖曳時優化：禁用 transition、固定 placeholder 高度、延遲展開父任務

**核心函數：**
- `findTaskById`：遞迴查找指定 ID 的任務
- `removeTaskFromTree`：從樹中移除任務
- `addChildToTask`：添加子任務到指定父任務
- `adjustTaskLevels`：調整任務層級
- `isChildTask`：判斷是否為子任務

### 3.2 任務重複性系統（Repeat System）

**功能描述：**
這是一個**複雜且完整的重複任務管理系統**，支援任務按設定的間隔自動重置。

#### 3.2.1 重複設定
- **時間單位**：支援「分鐘」和「天」兩種最小單位
- **間隔設定**：可設定重複間隔（如每 3 天、每 30 分鐘）
- **基準點**：可選擇以「開始日期」或「截止日期」作為重置基準
- **重置規則**：
  - 任務完成狀態重置為 `pending`
  - 進度重置為 0
  - 子任務遞迴重置（保留原始狀態類型）
  - 暫停狀態的任務跳過重置

#### 3.2.2 重置引擎（Reset Engine）
系統在**三個時機點**檢查並執行重置：
1. **啟動時檢查**：應用程式啟動時檢查所有重複任務
2. **定時輪詢**：每分鐘檢查一次（針對分鐘級重複）
3. **標籤切換**：進入任務詳情或甘特圖視圖時即時檢查

**重置邏輯：**
```javascript
// 核心重置函數：resetTaskTreeIfNeeded
- 計算當前時間窗口（windowStart）
- 判斷是否需要重置（shouldResetTask）
- 執行 backfill（補記）邏輯
- 遞迴重置子任務樹
```

#### 3.2.3 重複日誌系統（Repeat Log）
- **日誌記錄**：每天/每分鐘記錄任務完成狀態和進度
- **日誌保留**：無限期保留（使用者可手動刪除）
- **Backfill 機制**：最多補記最近 7 天的日誌（避免一次性回補過多）
- **日誌視覺化**：
  - 日誌列表：顯示最近 7-30 天的記錄
  - 日誌分析：包含完成率統計、趨勢圖表
  - 手動刪除：可刪除特定日期的日誌記錄

**日誌資料結構：**
```javascript
repeatLog: {
  "2025-01-15": {
    completed: true,
    progress: 100,
    maxProgress: 100
  }
}
```

### 3.3 文字膠囊動態任務生成系統

**功能描述：**
這是一個**創新的任務生成系統**，允許使用者透過「文字膠囊」和「任務模板」批量生成結構化的子任務。

#### 3.3.1 文字膠囊（Text Capsules）
- 使用者可以為任務建立多個文字膠囊（如：「書房」、「客廳」、「臥室」）
- 每個膠囊可以獨立編輯和刪除
- 膠囊狀態按任務獨立儲存

#### 3.3.2 膠囊任務模板設計器（Capsule Task Template Designer）

**核心功能：**

1. **模板結構設計**
   - 支援多層級任務結構設計
   - 任務名稱可包含固定文字和膠囊佔位符
   - 使用 `contentEditable` 實現混合內容編輯（文字 + 膠囊標籤）

2. **膠囊佔位符系統**
   - 顯示為藍色標籤樣式（「膠囊」）
   - 可在任務名稱中任意位置插入
   - 支援在文字間拖曳移動膠囊位置
   - 生成任務時，佔位符會被實際膠囊文字替換

3. **任務拖曳重組**
   - 任務節點可拖曳成為其他任務的子任務
   - 拖曳到任務上半部：成為同層級任務
   - 拖曳到任務下半部：成為子任務
   - 拖曳失敗時自動回退到原位置

4. **模板載入**
   - 可從預設模板載入（使用「任務模板」功能的所有模板）
   - 載入時自動計算任務層級（忽略模板原有層級）

5. **任務生成**
   - 每個膠囊生成一組獨立的任務樹
   - 自動替換模板中的膠囊佔位符
   - 生成的任務標記來源任務和膠囊 ID

**技術實現亮點：**
- 使用 `nameParts` 陣列結構儲存混合內容（文字 + 膠囊）
- `parseNamePartsFromElement` 和 `renderNamePartsToElement` 實現 DOM 與資料結構的雙向轉換
- 使用 `document.createTreeWalker` 解析 `contentEditable` 元素
- 膠囊拖曳使用 HTML5 Drag and Drop API，配合 `document.caretRangeFromPoint` 定位插入點

### 3.4 任務模板系統

**功能描述：**
- 預設內建「**生活環境整理**」模板
- 模板包含完整的任務樹結構和描述
- 可從模板快速建立專案任務
- 模板支援「膠囊標記」（`hasCapsules`），標記需要填入膠囊的任務
- 模板可儲存到 `localStorage`，支援多個模板

### 3.5 任務詳情管理

**功能描述：**
- **富文本編輯器**：使用 ReactQuill 實現任務描述編輯
- **日期時間選擇器**：開始日期、截止日期、時間設定
- **提前提醒系統**：可設定多個提醒（支援年/月/日/小時/分鐘單位）
- **地點選擇器**：整合 Google Maps API
- **圖示選擇器**：預設圖示庫 + 自訂圖示上傳
- **手繪畫布**：DrawingCanvas 元件支援手繪功能（多工具：筆、套索、文字、橡皮擦、圖片）
- **超連結管理**：可新增多個外部連結
- **集氣條（Cheer Up）**：任務激勵系統，可新增集氣項目並追蹤進度
- **可自訂布局**：支援拖曳重新排列任務詳情頁面的區塊

### 3.6 通知系統

**功能描述：**
- **瀏覽器原生通知 API**：使用 `Notification.requestPermission()` 請求權限
- **任務到期提醒**：任務到期時自動通知
- **提前提醒**：可設定多個提前提醒（支援年/月/日/小時/分鐘單位）
- **通知歷史記錄**：記錄所有通知，可查看歷史
- **通知權限管理**：自動請求通知權限
- **音效提醒**：通知時播放音效（`notification.mp3`）

### 3.7 任務標籤系統

**功能描述：**
- 任務可以添加多個標籤
- 標籤顏色自訂（可選擇顏色）
- 按標籤篩選任務
- 標籤管理：新增、編輯、刪除標籤

## 四、視圖系統

### 4.1 任務詳情視圖（Details）

**功能：**
- 完整的任務資訊編輯
- 可自訂布局（布局編輯模式）
- 支援拖曳重新排列區塊順序
- 所有任務屬性編輯（標題、描述、日期、地點、圖示等）

**布局區塊：**
- 任務標題和圖示
- 富文本描述
- 日期時間（開始/截止）
- 提醒設定
- 地點選擇
- 手繪畫布
- 超連結
- 集氣條
- 子任務列表

### 4.2 任務概覽視圖（Overview）

**功能：**
- 任務列表顯示（扁平化所有任務）
- **多條件篩選**：
  - 層級篩選（A/B/C/D/E/全部）
  - 狀態篩選（pending/completed/paused/全部）
  - 優先級篩選（high/medium/low/全部）
  - 標籤篩選
  - 搜尋關鍵字
- **多欄位排序**：
  - 標題（A-Z / Z-A）
  - 創建時間（新到舊 / 舊到新）
  - 截止日期（早到晚 / 晚到早）
  - 優先級（高到低 / 低到高）
  - 層級（A到E / E到A）
  - 狀態（pending/completed/paused）
- **批量選擇和操作**：可選擇多個任務進行批量操作

### 4.3 標籤視圖（Tags）

**功能：**
- 任務標籤管理
- 標籤顏色自訂
- 按標籤篩選任務
- 標籤列表顯示所有標籤及其使用次數

### 4.4 甘特圖視圖（Gantt）

**功能：**
- **時間軸視覺化**：任務以橫條形式顯示在時間軸上
- **三種縮放級別**：
  - 日視圖：以天為單位顯示
  - 週視圖：以週為單位顯示
  - 月視圖：以月為單位顯示
- **任務條顯示**：
  - 顯示開始和結束時間
  - 跨多天的任務顯示為連續條
  - 多行任務分段顯示（每行一段）
  - 自動分層處理重疊任務
- **互動功能**：
  - 日視圖支援 hover 預覽和編輯
  - 可調整欄寬（使用者偏好儲存）
  - 任務條顏色根據顯示模式變化
- **任務顯示模式**：
  - 預設：統一顏色
  - 層級：根據任務層級（A/B/C/D/E）顯示不同顏色
  - 優先級：根據優先級（high/medium/low）顯示不同顏色
  - 自訂：使用者自訂顏色

### 4.5 日曆視圖（Calendar）

**功能：**
- **月曆顯示**：顯示當前月份的日曆
- **任務按日期分組顯示**：
  - 任務顯示在開始日期和截止日期
  - 跨多天的任務在每個日期都顯示
- **互動功能**：
  - 點擊日期查看當日任務
  - 可切換月份（上一個月/下一個月）
- **任務條顯示**：
  - 從開始日期到截止日期顯示為連續條
  - 多個任務自動分層顯示
  - 任務條顯示任務標題（過長時淡化）

## 五、互動功能

### 5.1 拖曳排序功能

#### 5.1.1 任務樹拖曳
- **拖曳任務**：在樹狀圖中拖曳任務重新排序
- **改變層級**：拖曳到不同父任務下自動調整層級
- **同層排序**：在同一父任務下拖曳改變順序
- **視覺回饋**：拖曳時顯示插入指示線（placeholder）
- **優化措施**：
  - 拖曳時禁用 transition，避免視覺跳動
  - 固定 placeholder 高度，避免版面跳動
  - 延遲展開父任務，避免拖曳過程中高度變化

#### 5.1.2 布局編輯拖曳
- **拖曳區塊**：在任務詳情視圖中拖曳重新排列區塊順序
- **布局儲存**：布局狀態儲存在任務資料中

#### 5.1.3 膠囊拖曳
- **拖曳膠囊**：在任務名稱中拖曳膠囊標籤改變位置
- **精確定位**：使用 `document.caretRangeFromPoint` 定位插入點

#### 5.1.4 首頁任務拖曳
- **簡單任務列表**：在首頁（Home.js）中拖曳任務重新排序

### 5.2 點擊互動

- **任務選擇**：點擊任務標題選擇任務
- **展開/收起**：點擊展開圖標展開/收起子任務
- **編輯任務**：點擊編輯按鈕進入編輯模式
- **刪除任務**：點擊刪除按鈕刪除任務（需確認）
- **標籤切換**：點擊標籤按鈕切換視圖

### 5.3 鍵盤快捷鍵

- **Enter**：在新增任務輸入框中按 Enter 新增任務
- **Escape**：關閉模態框

### 5.4 表單互動

- **日期時間選擇**：使用 DateTimePicker 選擇日期和時間
- **圖示選擇**：使用 IconSelector 選擇圖示
- **地點選擇**：使用 MapPicker 選擇地點（整合 Google Maps）
- **富文本編輯**：使用 ReactQuill 編輯任務描述
- **手繪功能**：使用 DrawingCanvas 繪製圖形

## 六、響應式設計

### 6.1 響應式斷點

應用程式使用以下媒體查詢斷點：

```css
/* 平板設備 */
@media (max-width: 1024px) {
  /* 平板樣式調整 */
}

/* 手機設備（橫向） */
@media (max-width: 768px) {
  /* 手機樣式調整 */
  .capsules-grid {
    grid-template-columns: repeat(2, 1fr);
  }
}

/* 手機設備（直向） */
@media (max-width: 480px) {
  /* 小螢幕樣式調整 */
  .capsules-grid {
    grid-template-columns: 1fr;
  }
}
```

### 6.2 響應式布局

#### 6.2.1 主要容器
- **桌面**：左右分欄布局（側邊欄 + 內容區）

## 九、效能優化

### 9.1 狀態管理優化

- **useMemo**：計算衍生資料（如篩選後的任務列表、日曆任務、甘特圖任務）
- **useCallback**：穩定函數引用，避免不必要的重新渲染
- **條件渲染**：只在需要時渲染複雜元件（如甘特圖、日曆）

### 9.2 資料持久化優化

- **Debounce 機制**：使用 Lodash debounce 延遲 localStorage 寫入（避免頻繁寫入）
- **批量更新**：合併多個更新操作，減少寫入次數

### 9.3 渲染優化

- **虛擬化**：大型列表使用虛擬滾動（未來可考慮）
- **懶加載**：只在需要時載入複雜元件
- **CSS 優化**：使用 `will-change` 和 `transform` 優化動畫效能

### 9.4 拖曳優化

- **拖曳時禁用 transition**：避免視覺跳動
- **固定 placeholder 高度**：避免版面跳動
- **延遲展開父任務**：避免拖曳過程中高度變化
